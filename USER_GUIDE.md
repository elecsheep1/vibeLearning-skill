## 一、这是什么？

`vibe-learning` 是一个**科目无关的 AI 辅导引擎**。你只需为每个学科准备一个简单的"科目配置文件"（例如 `高等数学.md`、`强化学习.md`），AI 就会自动采用对应的教学风格：

- **exam 模式**：考研/考试模式 → 错题诊断、出变式题、输出复习卡片+思维导图
- **professional 模式**：工程实践模式 → 调试代码、项目引导、架构建议、知识树生成
- **research 模式**：学术研究模式 → 论文解读、概念辨析、开放思考题

所有学习内容会自动组织为模块化目录结构（笔记 + 知识树），支持 LaTeX 数学公式和 XMind 思维导图。

## 二、安装与文件结构

### 1. 存放位置

Skill 文件需要放在 Claude Code 的 skills 目录下：

- **macOS / Linux**：`~/.claude/skills/vibe-learning-skill/`
- **Windows**：`%USERPROFILE%\.claude\skills\vibe-learning-skill\`

### 2. 目录结构

```text
vibe-learning-skill/
├── SKILL.md                 # 核心引擎（已配置好）
├── subjects/                # 科目配置文件夹
│   ├── 高等数学.md          # exam 模式示例
│   ├── 英语.md              # exam 模式示例
│   ├── 政治.md              # exam 模式示例
│   ├── 自动控制原理.md      # exam 模式示例
│   └── 强化学习.md          # professional 模式示例
└── templates/               # 输出模板
    ├── mindmap.md           # XMind 思维导图生成模板
    ├── mistake-analysis.md  # 错题分析卡片模板
    └── debug-card.md        # 调试卡片模板
```

> 不需要手动修改 `SKILL.md`，以后只需维护 `subjects/` 下的科目配置文件即可。

### 3. 学习产出的模块化目录

当你学习某科目并让 AI 生成笔记或知识树时，工作区会自动形成以下结构：

```text
<工作区>/
├── <主学习规划文档>.md
├── <科目名>/                  ← AI 自动创建
│   ├── 笔记/                  ← 按主题存放学习笔记
│   │   ├── <主题1>-学习笔记.md
│   │   └── <主题2>-学习笔记.md
│   └── 知识树/                ← 知识树大纲 + XMind 导图
│       ├── <科目名>知识树.md       ← 文本版，可搜索/版本控制
│       └── <科目名>知识树.xmind    ← XMind 可视化浏览
```

所有文件之间用**相对路径交叉引用**，知识树用 ⬜ → 🟡 → ✅ 追踪每个知识点的学习进度。

## 三、日常使用流程（最重要）

### 场景：整理一道高数错题

1. **拍照/扫描题目**
   用软件（推荐 **Mathpix** 或 **SimpleTex**）将印刷体题目识别成**清晰的图片**（或直接复制识别出的文字）。

2. **打开 Claude Code**
   启动 Claude Code（终端输入 `claude` 或双击应用）。

3. **发送消息**

   - 先告诉 AI 你要学什么：

     ```text
     学习高等数学
     ```

   - 上传题目图片，并附上你的错误过程（用语音输入或简短打字）：

     ```text
     这道积分题我写的是：∫ x e^x dx = ½ x² e^x + C  
     正确答案是 e^x (x-1) + C  
     整理错题
     ```

4. **AI 输出**
   AI 会直接给你：

   - 一张 **错题卡片**（知识点、错误类型、根因、正确思路、易错点、解题套路、变式题）
   - 一张 **思维导图**（`.xmind` 格式，可在 XMind 中打开浏览和编辑，同时生成 markdown 大纲作为文本版）
   - 询问是否保存到错题本

5. **保存结果**
   你可以把卡片内容复制到自己的笔记文件（例如 `错题集/2025-01-20-分部积分.md`），方便以后复习。

### 场景：学习新知识（如强化学习）

1. 告诉 AI 你要学习的内容：

   ```text
   我想学习强化学习的 MDP 框架
   ```

2. AI 会：
   - 在 `强化学习/笔记/` 下创建学习笔记（`.md`，含 LaTeX 公式）
   - 在 `强化学习/知识树/` 下创建知识树大纲（`.md`）和 XMind 导图（`.xmind`）
   - 在主学习规划文档中更新链接和进展日志

3. 后续学完一个知识节点，让 AI 更新知识树中的状态标记。

### 场景：遇到配置中没有的新错误

当 AI 诊断后，如果它发现这种错误在 `高等数学.md` 中尚未记录，会主动询问：

> "我注意到本次错误类型在当前科目配置中尚未收录。是否需要我生成一段配置内容，添加到 `subjects/高等数学.md` 中？"

- 你只需回复 **"是"** 或 **"好"**
- AI 会输出一段完整的配置片段，例如：

  ```markdown
  ## 可追加到"积分"章节的内容
  - **分部积分选u错误**：...  
  ### 新增诊断规则
  - 用户使用分部积分后积分变复杂 → 选u顺序错误
  ```

- 你复制这段内容，打开 `subjects/高等数学.md`，粘贴到对应章节末尾，保存即可。

> 整个过程不到 20 秒，以后 AI 就能自动识别这类错误了。

### 场景：想学习其他科目

1. 创建 `subjects/<科目名>.md`（参考已有配置的格式，设置合适的 `learning_mode`）
2. 在 Claude Code 中说：`学习 <科目名>`
3. AI 会自动切换对应的辅导模式，并按模块化目录结构生成学习内容。

### 场景：临时切换学习模式

如果你想用 exam 模式分析工程类科目中的公式推导错误（而非 professional 的调试模式），可以说：

> "虽然现在学习规控，但请用 exam 模式帮我整理其中的公式推导错误"

AI 会在本次会话中临时覆盖模式。

### 场景：生成或更新知识树

```text
帮我为强化学习生成一个知识树
```

或：

```text
我刚学完了 DQN，帮我更新强化学习知识树
```

AI 会同步更新 `.xmind` 和 `.md` 两个版本。

## 四、常见问题

### Q1：AI 没有自动输出卡片，而是长篇对话？

**A**：你可能没有明确说"整理错题"。请确保在提供错题后加上指令：`整理错题` 或 `diagnose`。

### Q2：上传的图片识别不准怎么办？

**A**：Claude 对**清晰印刷体**识别率很高。如果识别有误，你可以：

- 用 Mathpix 重新生成更清晰的图片
- 或者直接把识别出的文字粘贴到对话中

### Q3：如何让 AI 不保存错题本就提问？

**A**：当 AI 询问"是否保存到错题本"时，回复"不用"即可，不会影响分析结果。

### Q4：科目配置文件中的"诊断规则"怎么写？

**A**：最简单的方法：让 AI 帮你生成。每次遇到新错误时，同意 AI 的"是否添加配置"建议，它自动生成的片段就是标准格式。你也可以直接仿照已有的规则写。

### Q5：生成的 .xmind 文件怎么打开？

**A**：下载并安装 [XMind](https://xmind.cn/)，用 XMind 打开 `.xmind` 文件即可看到完整的可折叠思维导图，支持编辑、标记、导出。同时也生成了配套的 `.md` 大纲文件，可以直接在编辑器中查看和搜索。

### Q6：LaTeX 公式在笔记中不渲染怎么办？

**A**：笔记中的公式使用标准 LaTeX 语法（`$...$` 行内、`$$...$$` 块）。确保你的编辑器安装了 LaTeX 数学渲染插件（VS Code 推荐 `Markdown+Math` 或 `Markdown All in One`；Obsidian 原生支持）。

### Q7：如何评估这个 Skill 对我有没有帮助？

**A**：Skill 内置了评估标准。你可以每两周抽样几道错题，记录：

- AI 输出的卡片是否完整
- 错误类型分类是否正确
- 你根据诊断能否自己改对变式题
- 每次用时多少秒

如果你觉得某个方面不满意（例如根因分析太浅），可以补充科目配置中的"诊断规则"来改进。

## 五、进阶技巧

### 1. 语音输入错误过程

在手机或电脑上用语音输入描述错误步骤（"我第一步直接乘积积分了"），比打字快得多。

### 2. 利用模板自定义卡片格式

如果你希望卡片顺序不同或增加字段，可以修改 `templates/mistake-analysis.md`。Skill 会优先使用模板中的格式。

### 3. 将内容同步到笔记软件

- **错题卡片**是 Markdown 格式，可直接复制粘贴到 Obsidian / Notion 等任何笔记软件。
- **XMind 导图**（`.xmind`）可在 XMind 中直接打开，支持导出为 PNG/SVG/PDF。
- **知识树大纲**（`.md`）适合放入 Obsidian 等支持双向链接的笔记软件。

### 4. 定期维护配置文件

每个月花 10 分钟，把 AI 建议添加的配置片段整理到对应的分类章节，删除重复项。你的配置文件会越来越智能。

### 5. 自定义知识树结构

知识树的节点结构由你控制——告诉 AI 你的偏好即可：

```text
知识树里把"工程实践"放在第一位，理论放后面
```

AI 会按你的要求重新组织知识树的层次结构。
