# 强化学习

## 元信息
- learning_mode: professional
- sub_mode: qa
- description: 强化学习（Reinforcement Learning）——具身智能的核心技术栈。涵盖 MDP、值函数方法、策略梯度、Actor-Critic、离线 RL 等。

## 推荐实践流程
1. 理解 MDP 框架（状态、动作、奖励、转移概率、折扣因子）
2. 用 Gymnasium 环境跑通随机策略，理解环境交互接口
3. 实现 Q-Learning（表格型，如 FrozenLake）
4. 实现 DQN（Atari 或 CartPole）
5. 实现策略梯度 REINFORCE
6. 用 Stable-Baselines3 跑通 PPO/SAC 训练 MuJoCo 环境
7. 理解 reward shaping 和 domain randomization
8. 尝试在自定义机器人环境中训练策略

## 知识点关联图
- MDP → 前置：[概率论、马尔可夫链] → 后续：[Bellman 方程、动态规划]
- 值函数方法（Q-Learning/DQN） → 前置：[MDP、Bellman 方程] → 后续：[Actor-Critic]
- 策略梯度（REINFORCE/PPO） → 前置：[梯度下降、期望最大化] → 延伸：[TRPO、SAC]
- Actor-Critic → 前置：[值函数 + 策略梯度] → 后续：[SAC、TD3]
- 基于模型的 RL → 前置：[最优控制、系统辨识] → 延伸：[MPC+RL 混合]
- Sim-to-Real → 前置：[Domain Randomization、系统辨识] → 后续：[真实机器人部署]

## 核心概念/常见误区
- 核心概念：Exploration vs Exploitation、Credit Assignment、Delayed Reward、Sample Efficiency
- 常见误区：
  - Q-Learning 中 Q 表不收敛 → 检查学习率衰减和探索率衰减策略
  - DQN 训练不稳定 → 检查 Target Network 更新频率和 Experience Replay 大小
  - PPO 策略崩溃 → 检查 clip ratio 和 advantage normalization
  - reward shaping 过度 → 可能引入人类偏见，导致次优策略
  - Sim-to-Real gap 被低估 → 需要 domain randomization + 系统辨识
